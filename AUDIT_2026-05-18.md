## 0. Executive Summary

1. **A homepage V2 está visualmente boa** — paleta coerente (branco + `#1E4FBF` + `#FF7A1A`), hierarquia clara, layout limpo. É um upgrade brutal vs as 5 iterações anteriores no histórico git.
2. **A página `/portfolio` está completamente quebrada** — ainda usa CDN Unsplash, importa `Header.tsx`/`Footer.tsx` legacy (paleta antiga preta/dourada) e o `CursorDot`/`InfiniteMenu` que LAYOUT_PLAN marcava como "remover". Linkada no `sitemap.ts` e indexável. **É o issue P0 mais grave.**
3. **Faltam todos os requisitos legais PT obrigatórios**: link para o Livro de Reclamações, Política de Privacidade, Termos & Condições, Política de Cookies, banner de consentimento de cookies (RGPD). Apresentar ao cliente sem isto é dar-lhe um produto fora-da-lei.
4. **Dead code massivo**: 14 componentes não usados (PixelSnow, LightRays, FloatingLines, InfiniteMenu, CursorDot, Hero, Services, Method, Testimonials, LeadForm, FAQ, Header, Footer, SectionHeader, SectionWrapper) — somam ~150 KB de fonte. `globals.css` tem 250 linhas da paleta antiga *stone/gold/glassmorphism* que V2.tsx ignora completamente. Dependências `three`, `ogl`, `gl-matrix`, `framer-motion` continuam no `package.json` mesmo sem qualquer import em V2.
5. **Reviews / claims em modo "fake-it"** — `aggregateRating: 4.9, reviewCount: 127` no Schema JSON-LD vs `+120 clientes` no UI. Reviews hardcoded em layout.tsx (3 reviews com nomes inventados em datas de 2024). Google pode penalizar e dá risco regulatório (CNPD/CRP). **Não publicar até ter testemunhos reais.**
6. **Form de lead não submete a lado nenhum** — `onSubmit={(e) => e.preventDefault()}` na `LeadCTA` (V2.tsx:968). Sem webhook n8n, sem POST, sem feedback ao utilizador. A "máquina de aquisição" do SPEC.md não está ligada.
7. **Footer com placeholders críticos** — NIF `—`, Alvará `—`, email `grupocronograma@hotmail.com` (hotmail é red flag para B2C premium ticket-alto), `sameAs: []` (sem redes sociais). Marca não inspira confiança no scroll final.
8. **SEO base é bom** — Schema `HomeAndConstructionBusiness` rico, OG image presente, robots/sitemap configurados, headings hierárquicos. Mas `metadataBase: https://cronohomeservice.pt` aponta para domínio sem DNS configurado → `canonical` partido em produção Vercel.

**Score global do site: 6.2/10** (homepage 7.5, mas /portfolio + legal compliance + dead code puxam para baixo).
**Veredicto: NO-GO** para apresentar ao cliente final como está. Com 1-2 dias de trabalho focado nos P0, fica em GO.

### 2.3. ICP diferente — não forçar comparações

- **OSCAR:** marketplace B2C de serviços rápidos sub-€100 (limpeza €29.90, fuga €46.90). App, 30min, scale 15k reviews.
- **Crono:** empresa de obras integradas €1.890+ (ticket alto, projeto 2-4 semanas+). Sem app, sem geo-matching, "gestor único" é venda principal.

O que **NÃO transferir** do OSCAR:
- App download (Crono não tem app, nem precisa)
- "Em 30 minutos" (Crono é visita em 24-48h, obra em semanas)
- 33 cidades / 3 países (Crono = região Lisboa)
- Volume de reviews (15k) — Crono usa qualidade > quantidade

O que **transferir bem** (e já está feito): clarity de CTAs, trust signals visíveis, hierarquia de scroll, grelha de categorias com fotos.

## 7. Detalhes por Skill

### 7.1. Design critique

**Pontos fortes:**
- Paleta `#1E4FBF` (confiança) + `#FF7A1A` (acento quente) + branco — coerente em todo o site, sem cores fora-da-palette
- Hierarquia tipográfica clara: H1 68px → H2 48px → body 18px. DM Sans bem escolhido para legibilidade
- Espaçamento generoso: `py-20 lg:py-28` consistente entre secções, breathing room ok
- Sticky header com backdrop-blur + `bg-white/95` é a escolha certa para overlay sobre conteúdo
- ServicesCategoryGrid 13 itens em 7 colunas em desktop — design tight e funcional
- Mobile-first parece sólido pelos breakpoints: `sm:` `lg:` bem distribuídos; CTAs com `py-5 px-9` = ~64x52px touch target (acima do 44px mínimo iOS)

**Pontos fracos:**
- **Hero illustration:** logo dentro de gradient círculo dentro de outro círculo amarelo — visual genérico "agência criativa 2018". Devia ser foto real de obra (ou render 3D de casa moderna se não houver fotos reais).
- **Press band:** "Margres Roca CIN Grohe Pladur Weber" como texto bold em Tailwind — parece reclamação por reclamar. Precisa de logos reais (mesmo que SVG simples).
- **Tipografia desperdiçada:** `Cormorant_Garamond` importado em `layout.tsx:12` mas **nunca usado** no V2.tsx. Adiciona ~30KB ao bundle sem ROI.
- **Animações:** apenas `transition` + `hover:-translate-y-1`. Sem stagger entry, sem scroll trigger. SPEC §7 prometia "Framer Motion para entradas suaves" — não está implementado. Não é dealbreaker mas SPEC vs implementation diverge.
- **ServicesGrid horizontal scroll** (V2.tsx:594 `min-w-[880px]`) — em mobile força scroll horizontal sempre, mesmo quando devia caber. Pode ser intencional mas é UX-quirky para senhor mais velho do target.

| Check | Estado | Detalhe |

| Alt-text em imagens | ✅ | Todas as `<img>` têm `alt` — verificado V2.tsx 10+ ocorrências |
| Headings hierárquicos | ✅ | H1 único no Hero, H2 por secção, H3 dentro, H4 no footer. Sem skips. |

### 7.3. Code review

**God-component / organização:**
- `V2.tsx` tem 1106 linhas com 11 componentes inline. Devia ser split: `components/v2/{Header,Hero,ServicesCategoryGrid,PressBand,HowItWorks,ServicesGrid,BeforeAfterLight,ServicesGallery,Method,Reviews,FAQSection,LeadCTA,Footer}.tsx` + `components/v2/V2Home.tsx` como composer. Reduz cognitive load, permite tree-shaking parcial, viabiliza unit tests por componente.
- `"use client"` em V2.tsx:1 força client-side render do site inteiro. Apenas `Header` (estado `open`) e `FAQSection` (estado `openIdx`) precisam de client. Quebrar em server components + client islands ganha SSR + LCP.

**Type safety (regra TS coding-style):**
- ✅ Sem `any` visível no V2.tsx
- ✅ Tipos explícitos em `ServiceCard`, `ServiceCategory`, props
- ⚠ `const BRAND = { ... }` definido (V2.tsx:12) mas **nunca usado** — todos os hex hardcoded inline. Devia ser usado ou removido. Pior: dificulta theming/dark-mode futuro.
- ⚠ Inline hex repetido: `#1E4FBF` aparece ~30 vezes, `#FF7A1A` ~25 vezes, `#0B1E3A` ~20 vezes. Devia estar em Tailwind config como custom color (`brand-blue`, `brand-orange`, `brand-navy`) e usar `bg-brand-blue` em todo o lado.

**Performance:**
- `<img>` em vez de `next/image` em **todas** as imagens (verificar com `eslint-disable` repetido 8x em V2.tsx). Perde-se: auto WebP/AVIF, srcset, blur placeholder, lazy loading nativo. **Maior impacto P1 single fix.**
- `loading="lazy"` está em alguns `<img>` mas não em todos (hero illustration não tem — ok porque above-fold).
- Sem Suspense boundaries, sem dynamic imports — bundle vai monolítico.

## 8. Conversion funnel critique

**Jornada esperada:** Chega → Hero → ServicesCategoryGrid → PressBand → HowItWorks → ServicesGrid (preços) → BeforeAfter → ServicesGallery → Method → Reviews → FAQ → LeadCTA.

**Pontos de fricção:**
1. **Hero illustration fraca** — primeiro contacto visual é logo num círculo, não foto de obra. Quem chega via Google Ads pesquisando "remodelação cozinha" não vê cozinha = bounce risk.
2. **PressBand sem logos reais** — texto "Margres Roca CIN" pode parecer fluff; um senhor mais velho pode pensar "estão a inventar". Logos reais (mesmo grayscale) resolvem.
3. **ServicesGrid horizontal scroll** — em mobile usa `min-w-[880px]` (V2.tsx:594) força scroll horizontal sempre. Confuso para utilizadores menos técnicos. Devia ser carousel com `snap` ou empilhar verticalmente.
4. **Method secção dark** — quebra o ritmo branco-azul até aí. É intencional (estilo OSCAR não tem este pattern) mas pode ser jarring. Test A/B se for crítico.
5. **LeadCTA form não submete** — pior fricção possível: utilizador converte, nada acontece, nem feedback. Abandona definitivamente.
6. **WhatsApp link sem pré-fill** — utilizador clica, abre WhatsApp, vê chat vazio, tem que pensar o que escrever, fricção alta.
7. **Sem multistep do SPEC** — formulário 2-campo é versão "lite". SPEC §7 prevê multistep com upload de foto + zona + urgência. Diferencial maior abandonado.

**Drop-off zones estimadas (sem analytics, hipóteses):**
- 30% bounce no Hero (illustration fraca + falta de Lisboa no H1)
- 15% scroll-fade entre ServicesCategoryGrid e HowItWorks
- 20% drop no ServicesGrid (scroll horizontal confuso)
- 10% engagement spike no BeforeAfter (jóia confirmada)
- 5% drop na Method (mudança de tema)
- 60% abandona no form (sem feedback)

**WhatsApp direct vs form:** WhatsApp aparece 4x (Hero, LeadCTA, Footer phone1, Footer phone2). Form aparece 1x. WhatsApp ganha por exposição. Bom para alto-ticket B2C — WhatsApp é mais íntimo + dá ao cliente sensação de "tenho contacto humano".
